# Mode, Oscillation, and Reversal

*A real-time tool to preserve correction in interaction*

---

## Purpose

To detect directional lock in conversation and restore oscillation through minimal reversal.

---

## Step 1 — Detect Lock

Ask:

- Am I repeating the same mode?
- Is certainty increasing?
- Is questioning decreasing?

**If yes → directional lock is likely**

---

## Step 2 — Identify Current Mode

- Regulation → calming / easing  
- Bonding → connecting / aligning  
- Validation → affirming  
- Narrative → explaining / framing  
- Diagnostic → analyzing / testing  

---

## Step 3 — Apply Reversal (one line only)

Introduce the missing counter-mode:

- **Diagnostic → Validation / Regulation**  
  → “That does make sense from your perspective”  
  → “Let’s slow this down a bit”

- **Validation / Bonding → Diagnostic**  
  → “What might be wrong here?”  
  → “What are we missing?”

- **Narrative → Diagnostic**  
  → “Where could this break?”  
  → “What doesn’t fit this explanation?”

- **Regulation → Narrative / Diagnostic**  
  → “What’s actually going on here?”  
  → “Let’s define the problem clearly”

---

## Step 4 — Check Effect

Ask:

- Did the interaction open up?
- Is there more space to think?

- ✔ Yes → oscillation restored  
- ❌ No → apply a slightly stronger reversal  

---

## Pocket Version

- Stuck in one mode?  
- Name it  
- Add one opposite line  
- Did it open?

---

## Core Principles

> If the mode doesn’t change, correction doesn’t happen.

> Correction requires directional change.  
> Reversal provides that change.

---

## Summary

Directional lock → loss of correction  
Reversal → restored oscillation  

Maintain movement between modes to preserve coherence.
