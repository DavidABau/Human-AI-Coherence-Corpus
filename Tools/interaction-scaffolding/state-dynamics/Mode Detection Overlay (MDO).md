# Mode Detection Overlay (MDO)

**Purpose:**
Apply the conversational mode framework to **internal states**, allowing the same system to be used for:

* external interaction (people)
* internal interaction (self)

---

## Core Principle

> **Internal states express themselves in modes, just like conversations do.**

---

## The Five Modes (Internal Form)

### 1. **Regulation Mode**

* Goal: stabilise system
* Internal signals:

  * overwhelm
  * fatigue
  * stress spikes
* Felt as:

  * “I need to slow down / settle”

---

### 2. **Bonding Mode**

* Goal: connection / belonging
* Internal signals:

  * loneliness
  * desire for closeness
  * emotional openness
* Felt as:

  * “I want connection / warmth”

---

### 3. **Validation Mode**

* Goal: recognition / being seen
* Internal signals:

  * frustration
  * hurt
  * self-doubt
* Felt as:

  * “This matters / I need this acknowledged”

---

### 4. **Narrative Mode**

* Goal: meaning-making
* Internal signals:

  * rumination
  * storytelling
  * interpretation loops
* Felt as:

  * “What does this mean?”

---

### 5. **Diagnostic Mode**

* Goal: analysis / problem-solving
* Internal signals:

  * pattern recognition
  * system thinking
  * detachment
* Felt as:

  * “What’s actually going on?”

---

## Overlay Concept

```text
External Conversation Modes  ↔  Internal State Modes
```

Same structure:

* different domain

---

## Mode ↔ SSM Mapping

| Mode       | Typical Signal Type    | Steering Risk             |
| ---------- | ---------------------- | ------------------------- |
| Regulation | overload / instability | collapse if ignored (LCM) |
| Bonding    | connection need        | misdirected dependency    |
| Validation | recognition need       | amplification of distress |
| Narrative  | meaning construction   | rumination loops          |
| Diagnostic | analysis               | over-detachment           |

---

## Mode Detection (Internal)

Ask:

> **“What mode is this state trying to operate in?”**

This replaces:

* vague “how do I feel?”

with:

* structured detection

---

## Mode-Appropriate Steering

### Regulation Mode

→ reduce load
→ stabilise first (LCM)

---

### Bonding Mode

→ seek or simulate connection
→ avoid isolating analysis

---

### Validation Mode

→ acknowledge before solving
→ “this matters” internally

---

### Narrative Mode

→ limit loop duration
→ shift if repetitive

---

### Diagnostic Mode

→ apply selectively
→ avoid overriding other modes

---

## Failure Pattern

> **Using the wrong mode internally**

Examples:

* using Diagnostic when Regulation is needed
* using Narrative when Validation is needed

Result:

* increased instability
* loop reinforcement

---

## Integration with SSM

```text
Signal → Mode Detection → Mode-Appropriate Steering → Reinforcement
```

---

## Integration with LCM

* High load → default mode dominance
* Often:

  * Regulation ignored
  * Narrative or Reactive modes take over

---

## Key Insight

> **Internal misalignment of modes produces the same failure patterns as external miscommunication.**

---

## Compressed Form

> **Detect the mode → match the response → avoid mismatch**

---

## Notes

* Modes are not mutually exclusive
* One mode is usually dominant at a given moment
* Correct sequencing matters (e.g. Validation → then Diagnostic)
